# 2026-07-21

Direction day — no code. Decided the **v2 repositioning**: BrainCue widens from
interview copilot to **universal ambient conversational companion** — it sits in
your real conversations and activities (interviews on either side, meetings,
tutoring, ambient companionship while working/gaming) and contributes through
the overlay or voice. Interviews become one mode of a shared engine.

**Why this framing (and not "voice mode clone"):** generic voice chat is
ChatGPT/Gemini turf. BrainCue's structural moat is what a cloud chat app can't
do — loopback presence inside real calls, a capture-invisible overlay, local
grounding in the user's corpus, BYO key. Positioning: "the AI in the room with
you"; voice is a surface, not the product.

**Key insight from the code audit:** the universal primitives largely exist —
realtime STT, question classifier (= a reactive trigger policy), RAG, streaming
answers, TTS, and a full two-way voice loop in sparring/mock. The pivot is
engine extraction + repositioning, not a rebuild.

## Decisions

- Keep the **BrainCue** name and appId; retire the "interview copilot"
  descriptor. "Cue Card" stays.
- Interview mode is the regression gate — no refactor may degrade it.
- Companion/game-buddy is deliberately **last** (needs memory, interjection
  policy, realtime voice, cost governance).
- Mock/sparring migrate onto the engine's dialogue loop **in Phase 3** (with
  Tutor), not during the Phase 1 extraction.

## Prompt 0 + Prompt 1 — audit, plan, baseline guardrails (`feat/v2-baseline`)

**Prompt 0 (audit/plan, no code):** 3-agent subsystem audit (AI services,
renderer surfaces, DB/IPC) + direct reads of types/ipc/schema/sessionManager.
Full plan condensed into `12-ENGINE-PLAN.md`. Highlights: `answerCards.ts` is
already ~generic; `client.ts`+`models.ts` are the provider seam; enums
duplicated across 4 ipc files; `profiles.answer_style` dead; `fkRebuild.ts`
carries duplicate sessions/chunks DDL; migration runner has no down path (→
boot-time DB backup lands in Prompt 2); ~20 doc-drift items catalogued.

**Prompt 1 (this branch):**
- `package-lock.json` root metadata repaired (1.5.2 → 1.5.3, no version bump).
- **`src/main/test/dbHarness.ts`** — real drizzle migrations on in-memory
  **sql.js** (new devDep): better-sqlite3 is rebuilt for Electron's ABI so it
  can't load under vitest, which is why no test touched the DB before. The
  harness takes an optional fixture image — Prompt 2's migration tests plug in
  there.
- **`src/main/test/architecture.test.ts`** — process-boundary contract:
  renderer/preload/shared may never import electron(main-only)/openai/
  better-sqlite3/drizzle/ws or reach into `src/main` (type-only imports
  exempt); built renderer bundle must not contain `api.openai.com`/
  `apiKeyStore`/`better_sqlite3`/`safeStorage`/`getDecrypted`.
- **`sessionManager.parity.test.ts`** — pins the live pipeline before the
  Prompt-3 extraction: start (row/transcriber/overlay seeding), final-
  transcript persistence, classifier gate (not-a-question + <0.4 floor),
  retrieval args, streamed event ORDER (question → context → deltas → done),
  answer persistence w/ tokens, stop/cleanup (+ dead pipeline after stop),
  and the stale-abort race: an aborted stream that wakes late can neither
  overwrite the newer regeneration nor wedge the answer slot.
- Runtime behavior: zero changes. **216 tests green** (204 + 12), typecheck +
  build clean.
