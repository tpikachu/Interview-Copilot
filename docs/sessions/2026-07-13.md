## Interview-UX pass (commit `4d85663`)

Five user-requested items:

1. **Question history ON by default** — the Cue Card keeps every question; older cards
   auto-collapse when a new one arrives (expand on click). History toggle still there.
2. **Inline pronunciation** — respellings now injected *right after the word in context*
   (`injectPronunciations` in `overlay/pronunciation.ts`) instead of the below-answer
   panel (removed). First occurrence per word, case-insensitive; conditional word
   boundaries so symbol-words like "C++" match. Copy/persist keep the clean body.
3. **Coding 4-beat structure** — `codingRules(language, format)` now prompts
   **Understanding → Plan → Solution → Evaluation** ("So this problem really comes down
   to…"), with per-format delivery (`CODING_DELIVERY`: terse for key_points, spoken
   walkthrough for explanation/story_teller, thorough for detailed). The live format is
   read from `sessionManager.activeAnswerFormat()` at solve time (fallback
   `'explanation'`); new `codingPrompt.test.ts`.
4. **SWE-only interview types** — `product` and `sales` removed from `InterviewType`
   and every selector (`QuestionType` keeps `'product'` for old rows).
5. **Answer application questions later** (Tailor Resume) — questions are optional at
   tailor time; ApplicationModal gains an "Answer application questions" section
   (one per line) → new `applications:answer-questions` IPC → `answerApplicationQuestions`
   in `tailor.ts` (grounded-only, first person) → `applicationsRepo.appendAnswers`
   merges into the stored JSON. Model call first, write after.

## Whole-project audit

6 dimension auditors + refute-verify over the entire codebase: **11 confirmed defects**
(6 further claims refuted) + a 32-item improvement backlog. Coverage caveat: session
rate limits killed the **ipc-db** and **dashboard-security** auditors and several
verifies — those dimensions are still owed a re-run. User approved fixing #1–#6 + two
quick lows (#8, #10).

## Audit fixes (this commit)

1. **(high) Concurrent answer streams corrupted one card** — the overlay kept a single
   pending-token buffer + `streamingId`, so a live-session answer and a coding solve
   streaming together interleaved text into whichever card flushed last. Fixed on both
   sides: the overlay buffers **per questionId** (`Map`, flush reduces every entry via
   `appendById`; `answer-reset` drops only its own stream's buffer), and `codingMode.ts`
   **serializes solves** — `streamToOverlay` takes a generator *factory*, keeps a
   module-level `AbortController`, aborts the previous solve when a new one starts
   (aborted streams skip the error broadcast, still emit `answerDone`).
2. **Answering gate ownership race** — `generateAnswer`'s `finally` cleared
   `live.answering` unconditionally, so an aborted-and-replaced stream could reopen the
   gate under the replacement. Both clears now guarded by `live.answerAbort === abort`.
3. **Mock start failure leaked a half-started session** — `mockManager.start()` now
   guards on `apiKeyStore.isPresent()` up front and wraps the first `ask()` in
   try/catch → `sessionManager.stop(id)` (deletes the mock row, hides the overlay) +
   rethrow, instead of leaving a dead session row + overlay.
4. **Tailor calls could retry silently / hang** — both `responses.create` calls now pass
   `{ timeout: 300_000, maxRetries: 0 }` (a tailor is expensive; a silent SDK retry
   doubles cost, and the default timeout is too generous for a UI-blocking op).
5. **Reasoning model as the answer override → blank cards** — reasoning models burn
   hidden tokens against `max_output_tokens` first, so the tight per-format ceiling
   could be consumed before any visible text. `answer.ts` now detects
   `isReasoningModel(model('answer'))` → sends `reasoning.effort` (override or `low`),
   adds +1024 token headroom, and **throws** "returned no text" instead of leaving a
   silently blank card (surfaces as a sessionError toast).
6. **`saveCoding` clobbered concurrent settings edits** — the overlay wrote its coding
   model/effort by spreading a *stale ref* of the settings maps. Now an async
   read-modify-write: fetch fresh settings, merge only the coding key, write.
7. **(#8) Mock history counted unspoken questions** — `ask()` pushes to `mock.history`
   only after `speak()` succeeds (a TTS failure no longer consumes one of the N
   questions); `answerQuestion` still fire-and-forgets before speak for parallelism.
8. **(#10) `isReasoningModel` flagged chat-tuned GPT-5s** — `gpt-5-chat-latest` etc.
   are non-reasoning despite the prefix; sending `reasoning.effort` to them errors.
   The regex now excludes `/chat/i`.

**Adversarial review of the fix diff** (2 dimensions → refute-verify; 3 confirmed,
1 refuted — all three fixed in this same commit):

- *(med)* **Fix 2 was incomplete** — `processFinalTranscript` claims the answering slot
  before the classify await, and its not-a-question / catch branches released it
  *unconditionally*. A manual Ask/regenerate during that ~0.5s window takes ownership
  (sets `answerAbort`); the stale classify then reopened the gate under the streaming
  answer, letting the interviewer's next question abort the user's manual answer. Both
  releases now skip when `live.answerAbort` is set (a stream owns the slot; its own
  finally releases), and the catch also re-checks `sessionId`.
- *(low)* **Fix 3's teardown crashed on a concurrent user Stop** — the overlay is live
  during the multi-second first `ask()`, so Stop can delete the mock row first;
  `sessionManager.stop()` then TypeError'd on the missing row *from the catch*, masking
  the original error and skipping `mock = null`. `stop()` now throws a clean
  "Session not found" on a gone row; the start-catch and `end()` treat stop as
  best-effort (try/catch) so `mock` is always released and the real error surfaces;
  start's success path reports "Mock interview was stopped" instead of a TypeError.
- *(low)* **Fix 6's mirror image** — dashboard "Save models" wrote the *whole* override
  map from page-load state, silently wiping a `models.coding` override the Cue Card
  panel saved while Settings sat open. ModelsCard now tracks *dirty* keys and `save()`
  does a fresh read-modify-write applying only those (empty = delete = "use default");
  preset switch / Reset keep their explicit clear-all semantics.

Verified after review fixes: `typecheck` · 154 unit (+4) · `build` green.

## Closing the audit coverage gap (dashboard-security + ipc-db)

The two dimensions the original audit lost to rate limits were re-run
(dashboard-security via agents; ipc-db inline after agents rate-limited again).

**dashboard-security — 1 confirmed (med), fixed:** `fetchUrlText` buffered the ENTIRE
HTTP body into main-process memory before applying the 3 MB cap, and the 15s abort
timer was cleared as soon as headers arrived — so a pasted job/company URL (or a
shortener redirect) streaming a huge body could OOM the main process (killing a live
interview). Reachable via Tailor "Fetch" and job-save company research (loops up to 6
URLs). Fixed: one deadline now covers headers AND body; the body is read via streaming
with the byte cap enforced mid-download (early break cancels the stream). Also closed
the unverified-but-real low: `redirect: 'follow'` let a public-looking link land on
localhost/private/metadata hosts whose content would then be embedded into OpenAI
prompts — redirected responses now re-check the FINAL url against a literal
private-host list (typed intranet URLs remain allowed; only the redirect disguise is
blocked). New `fetchUrl.test.ts` (+7: private-host matrix, endless-body cap, redirect
guard). Refuted (defense-in-depth only, no reachable trigger): `setWindowOpenHandler`
lacks a scheme allowlist — the renderer has zero anchors/window.open and Markdown
renders links as non-clickable spans.

**ipc-db — inline audit, clean:** `PRAGMA foreign_keys = ON` at open; every handler
goes through `handle()` (zod + Result envelope, normalized errors); pagination bounded
(`limit ≤ 100/200`, `offset ≥ 0`); LIKE search parameterized (only cosmetic: `%`/`_`
in the query act as wildcards); the dev DB explorer validates table names against
`sqlite_master` and is dev-only; preload↔main channels have no drift; events use
targeted `webContents.send` correctly; drizzle journal ↔ 0000–0006 SQL consistent.
Backlog notes: clear-and-reinsert index passes (`reindexProfile`/`indexJob`) aren't
transactional (crash window is small and self-heals via Re-index); `shortcutFired`
is a dead event (declared + exposed, never sent/consumed); LIKE wildcard escaping.

Verified: `typecheck` · 161 unit (+7) · `build` green.
