## Prompt 4 — provider capability seam (`feat/v2-providers-seam`)

The engine now depends on **capability interfaces**, not the OpenAI SDK.
Design line held throughout: interfaces carry TRANSPORT only (delta/usage
events, JSON round-trips); prompt building, format ceilings, the `meta` event,
and no-text errors stay in the service modules — so a second provider is a
transport swap, not a rewrite.

- **`src/main/providers/`**: `types.ts` (Chat/Embedding/RealtimeStt/BatchStt/
  Speech/Vision interfaces + `EmbeddingIdentity`), `errors.ts`
  (`CapabilityUnavailableError` with user-safe per-capability copy,
  `EmbeddingIdentityMismatchError`), `registry.ts` (per-capability selection,
  default OpenAI everywhere; registration is data so tests inject fakes),
  `openai/index.ts` (the reference impls wrapping the existing modules —
  reasoning-effort param + 1024-token reasoning headroom moved HERE from
  answer.ts, byte-identical request bodies).
- **Call sites switched**: answer/questions/followup → `chat.stream`/`chat.json`;
  retriever + indexProfile → `embedding` (+ identity guard on every write
  path); engine sourceAdapter → `realtimeStt.open` (provider returns a started
  session); engine.ingestAudio → `batchStt`. parsing/brief/stories/tailor/
  interviewer/feedback/coding/vision keep direct SDK calls — migrate
  opportunistically (documented in 06).
- **Migration 0010**: `embeddings.provider` (default 'openai').
  `rag/embeddingIdentity.ts` refuses mixed embedding spaces; re-index
  requirement is now explicit in the domain (message tells the user what to
  do), guided re-index UI deferred per the prompt.
- **Behavior preserved**: existing answer/followup/questions tests pass
  UNCHANGED in their assertions (mocks intercept the same resolved
  client/models modules through the provider). One deliberate v1-split kept:
  followup swallows only SyntaxError (malformed prediction), API errors still
  propagate to the fire-and-forget log.warn. One micro-difference noted: when
  a stream yields no text, the usage event now precedes the thrown error
  (previously neither was emitted) — no persistence/user-facing change.
- **Test-mock ripple**: modules importing the registry transitively load
  providers/openai, which imports the full models surface + realtime (whose
  apiKey→env chain needs electron). Four test files' mocks extended
  (EMBEDDING_DIM + a RealtimeTranscriber stub); parity's client mock gained a
  throwing `openai()` so nothing can reach the network.
- **New tests**: registry resolution for all six capabilities, embedding
  identity from config, capability-gap error copy, **streamAnswer end-to-end
  through a registered fake ChatProvider** (the engine-on-fakes proof),
  identity-guard accept/refuse cases, fixture-migration provider stamp.

**235 tests green** (229 + 6), typecheck + build clean. No version bump; no
second provider (Prompt 11); no UI changes.

## Prompt 5 — ContributionCard + Overlay decomposition (`feat/v2-contribution-cards`)

The 1,337-line `Overlay.tsx` is now a ~450-line shell; the Cue Card renders
**generic Contributions** (all 11 kinds) instead of hard-coded answer cards.
Interview cards render pixel-identically — the registry just makes "answer" one
kind among many.

- **Generic contribution events** (`EVENTS.contribution*`): `open` (id/kind/
  title), `delta` (token), `patch` (named `meta`/`context`/`followup` fields —
  the v1 payloads verbatim), `done`, `reset`. Emitted by
  `src/main/ipc/contributionBridge.ts`, which **dual-emits** each one alongside
  its legacy `answer-*`/`question-detected` twin with the byte-identical v1
  payload — the compat adapter for one release. engineSession + codingMode now
  call the bridge; nothing broadcasts answer events directly anymore. Kind
  mapping at the source: question `type === 'coding'` → `code`, else `answer`.
- **Renderer structure** (`src/renderer/overlay/`): `Overlay.tsx` (shell:
  window state, subscriptions, composition), `store/useOverlayStore.ts`
  (zustand card feed; pure reducers in `cards/model.ts`),
  `lib/streamBuffer.ts` (the per-contribution rAF token coalescer, extracted),
  `cards/` (ContributionCard frame + `registry.ts` + per-kind views +
  `UnknownCardView` fallback), `controls/` (HeaderBar, SessionBar,
  AnswerControls, AskBar, Btn, EqualizerBars), `panels/` (ClientNotes,
  Captures, ErrorBanner, AudioMeter, Transcript, DataSent, SettingsModal).
  `answerCards.ts` is deleted (superseded by `cards/model.ts`).
- **Capabilities, not boolean soup**: the registry declares per-kind
  `{ copy, regenerate: { tooltip } | false }` and a chip label; the card frame
  renders actions from those. answer/code keep the v1 "Q:" header; the
  regenerate handler is uniform (session pipeline → falls back to re-solve),
  preserving the coding-card semantics exactly. Unknown future kinds render a
  dashed fallback card with the raw kind as chip — never a crash. Card views
  are PURE (callbacks in, no `window.api`) so they smoke-test via
  `react-dom/server` without jsdom.
- **The overlay consumes ONLY the generic events now**; the dashboard
  (`useLiveSession`) still uses the legacy events, which the bridge keeps
  byte-identical (pinned by contributionBridge.test.ts + two new parity tests).
- **Frozen e2e selectors preserved**: `/Stop|Pause/` button labels (SessionBar
  comment marks them frozen), window sizes untouched, dashboard nav untouched.
  Privacy hard test not run (needs a display) — phase boundary item.
- **New tests (+41 → 276)**: card-model reducers, overlay store (incl.
  concurrent-stream interleave routing), streamBuffer lanes/drop/cancel,
  render smoke for every kind + collapsed/fallback/capabilities/annotations
  (`registry.test.tsx` via `renderToStaticMarkup`; vitest now includes
  `.test.tsx`), bridge dual-emit contract, and parity: legacy event ↔ generic
  twin equality on the full pipeline + regenerate. All 11 pre-existing parity
  tests pass unmodified.

**276 tests green** (34 files), typecheck + build clean. No version bump.

## Prompt 7 — Meeting Copilot (`feat/v2-meeting-mode`)

The first truly NEW engine mode — a configuration over the shared engine
(`modes/meeting.mode.ts` + `ModeDefinition.ambient`), zero interview-pipeline
forks (all parity tests pass unmodified).

- **Trigger architecture** (exactly the prompt's ladder):
  `trigger/meetingHeuristics.ts` (pure, deterministic first pass — greetings/
  filler skip WITHOUT a classifier call; explicit action items/decisions/
  questions detected by rules) → `trigger/salience.ts` (structured classifier
  for ambiguous turns only, task 'classify', **zod-validated; invalid/failed
  output = silence**) → `trigger/ambientPolicy.ts` (per-session state:
  per-kind confidence floors, global + per-kind cooldowns, normalized-title
  dedupe; **the model scores, code decides**). `trigger/presence.ts` maps
  Summoned/Quiet/Balanced/Active to EXPLICIT numbers — never a slider into a
  prompt. Warnings need ≥0.85 at every level. Hard pause wins (engine gate).
- **Open questions**: a question turn is HELD, not emitted; it matures into a
  card only after 2 substantive turns pass without an answering turn
  (conservative `answersQuestion` rules: content-word overlap / bare yes-no /
  answer-lead + number).
- **Cards**: action_item/decision/open_question/warning bodies QUOTE the
  transcript turn (can never claim more than was said; owners/deadlines only
  from explicit extraction). `context` cards only exist when retrieval finds
  chunks; body generated from ONLY those snippets (zod-gated `relevant`
  escape hatch), provenance in sourceRefs + chunks surfaced via the
  contribution patch → "data sent". New `decision` ContributionKind + overlay
  DecisionCardView. Ambient cards emit via `emitAmbientContribution` —
  **generic contribution events only, no legacy answer twins**.
- **Report** (`engine/meetingReport.ts`): summary/decisions/actionItems/
  openQuestions, zod-parsed, then `groundReport` — a deterministic
  post-filter that NULLS any owner/deadline not literally present in the
  transcript. Persisted as ONE `summary` contribution (meta.reportType
  'meeting'); generated eagerly at stop, get-or-generate on view; action
  items/open questions editable in the Sessions page's MeetingReportModal
  via `contributions:update`.
- **Plumbing**: `session:start` gains mode+presence (defaults preserve v1);
  engine mode registry (`modeFor`), resume restores the mode; meeting speaker
  is 'them' (v2 vocab; overlay interim label follows the stream's speaker).
  FLAGS.meeting flipped ON (Labs badge in start flow + Home card); presence
  picker (default Quiet) + meeting-aware transparency summary in the start
  modal. Audio: single-source capture preserved through the source adapter —
  dual mic+system capture is deliberately NOT pretended; bounded per prompt.
- **Acceptance suite** (`meeting.acceptance.test.ts` — deterministic fixture
  through the REAL engine on sql.js with scripted providers + fake timers):
  greeting/small-talk silence with the classifier never called; explicit
  action item caught once (duplicate suppressed); unanswered budget question
  matures into an open-question card; context card retrieves the $99/seat
  document fact with chunk provenance; decision card; hard pause wins; report
  grounded (invented "Bob"/"March 3rd" nulled, explicit "by Friday" kept);
  generic-only emission (zero questionDetected/answerDelta); summoned
  presence contributes nothing. Plus unit suites for heuristics, the policy
  gates, salience schema, and report grounding/rendering.

**317 tests green** (39 files; +34), typecheck + build clean. No version
bump. Parity gate: all pre-existing interview parity tests pass unmodified.

## Prompt 8 — trustworthy local memory (`feat/v2-memory`)

Memory belongs to the user: consent-gated capture, review-first lifecycle,
scoped recall with visible provenance — never an opaque profile.

- **Schema (0011)**: ONE lifecycle table `memories` (pending = candidate,
  approved = item, rejected/archived out of recall) with scope
  (`pack_id` null = global), category/confidence/importance/sensitive,
  provenance json, lastUsedAt/expiresAt — and the **embedding ON the row**
  (with its identity), so delete removes the vector atomically and memory can
  never leak into document retrieval. Plus `jobs.memory_enabled` (per-Space
  opt-out) and the `memory_enabled` settings consent key (default OFF).
- **Extraction** (`services/memory/extractor.ts`): fires at session stop but
  is a no-op before consent (model never called — tested); ≤5 zod-validated
  candidates, 0.6 confidence floor, and `sensitiveFilter.ts` REJECTS
  secrets/credentials/payment/gov-ID/health/sensitive-personal content before
  persistence. Everything lands `pending` with session provenance.
- **Review** (Library › Memory, FLAGS.memory ON): global consent switch,
  approve-with-edits (embeds via the provider + embedding-identity guard;
  sensitive edits refused — a paste can't smuggle a secret), reject, search,
  edit (content edits re-embed), archive, hard delete, per-Space toggles.
- **Recall** (`services/memory/recall.ts`): hybrid — cosine is the GATE
  (0.25 floor on the semantic score alone); lexical overlap, importance, and
  recency are small tiebreak weights, never substitutes. Scope-aware
  (global + active Space), approved-only, unexpired, identity-matched; top-3,
  300-char clip, lastUsedAt stamped; failures return [] (memory never breaks
  an answer).
- **Engine integration**: `generateContribution` recalls after `ground()`;
  memories ride `contextSent`/`contribution:patch` as a separate array (the
  Cue Card "data sent" panel shows a violet "Memory used [M#]" section);
  the answer prompt gains a MEMORY block cited as **[M1]…** (separate from
  document [n] numbers) ONLY when memories exist — with none, the prompt is
  byte-identical to v1 (pinned by the untouched answer tests). Contribution
  sourceRefs record every memory used.
- **Privacy engineering**: no cloud sync; wipe cascades through memories
  (FK test); deletion cascade is single-row by construction; encryption at
  rest is DESIGNED (07: safeStorage-wrapped AES-256-GCM data key, lazy
  dual-read migration, packaging-safe — deferred pending two UX decisions,
  per the prompt's "present a design first").
- **No proactive interjection**: recall only ever feeds question answering;
  the Companion behaviors stay in Prompt 10.
- **Tests (+29 → 346)**: sensitive-filter classes + benign passes; extraction
  gates (consent short-circuit, Space opt-out, invalid shape → nothing,
  floor + sensitive drops, pending + provenance); review lifecycle
  (approve embeds, edit re-embeds, sensitive edit refused, delete removes
  row+vector); scoped hybrid recall (approved-only/expiry/identity/scope,
  semantic floor vs importance, cap + clip, lastUsedAt, consent
  short-circuits); [M#] prompt block + byte-parity without memories;
  migration 0011 on the v1.5 fixture + profile-delete cascade.

## Prompt 9 — voice/summon layer (`feat/v2-voice`)

Voice is an OUTPUT SURFACE over the contribution pipeline, not a mode: one
push-to-talk key from anywhere, spoken replies with barge-in, and a
no-session quick ask — the engine still owns all generation and persistence.

- **Dialogue controller** (`services/voice/dialogueController.ts`): a PURE
  explicit FSM — idle/listening/thinking/speaking/interrupted/paused/error —
  with a declared transition table (undeclared moves are no-ops) and a turn
  `generation` counter bumped by SUMMON/CANCEL/INTERRUPT/PAUSE/FAIL. Every
  async hop in the orchestrator re-checks the generation, which is the whole
  stale-work story (late STT results, late TTS segments, queued audio).
- **Orchestrator** (`voiceService.ts`): summon is ONE state-dependent press —
  idle→listen, listening→send, speaking→barge-in, thinking→abandon+re-listen.
  PCM16 frames buffer in main only while `listening` (short-tap and 120 s
  guards), get WAV-wrapped (`wav.ts`) into the batchStt capability, and the
  transcript routes: live session → `engine.askActive` (the SAME direct-ask
  pipeline as the typed Ask box — v1 persistence, dual-emitted events),
  followed via a new main-process `bridgeTap` on the contribution bridge; no
  session → `quickAnswer.ts` over the default Space (prefs), streamed as
  GENERIC-only contribution events like ambient cards.
- **Streamed speech over a one-shot capability**: `sentenceStream.ts` chunks
  the token stream into speakable sentences (markdown/citations stripped);
  each synthesizes via the speech capability and broadcasts as a
  `voice:audio-segment` (generation + seq + empty `last` marker). Speech
  starts on the FIRST sentence while the model still writes. The overlay's
  `voicePlayer.ts` plays segments in order (setSinkId where supported) and
  reports playbackDone; any state change away from speaking or a generation
  bump silences it immediately.
- **Barge-in**: overlay capture stays open through the turn; sustained speech
  while BrainCue talks (renderer RMS VAD, echo-cancelled mic) →
  `voice:interrupt` → speaking→interrupted→listening with a generation bump —
  synthesis already in flight resolves and is DROPPED, never played (pinned).
- **Never speaks unprompted**: only summoned replies reach TTS — ambient
  cards are text-only by construction (pinned); hard mute (prefs) wins over
  everything and mid-speech mute cancels; a hard-paused session pauses voice.
  Muted / missing speech capability / synth failure → text-only fallback
  (reply still streams to the card, state notes `textOnly`).
- **Quick-ask persistence per user settings**: ephemeral by default; with
  `saveQuickAsks` on, contributions land in ONE reused per-profile
  `mode='companion'` session row (the FK home that keeps Sessions tidy).
- **Surfaces**: global shortcut `voice:summon` (Ctrl+Shift+T, rebindable);
  Cue Card `VoiceBar` (state pill, send/interrupt/cancel, hard mute) renders
  session-independent; SettingsModal voice section (voice, output device,
  quick-ask Space, save toggle); Home's "Talk to BrainCue" graduated from the
  Labs strip to a primary action. FLAGS.voice ON.
- **Privacy**: mic opens only for an explicit summon; STT/TTS run in main
  (key isolation unchanged); no audio/base64 ever logged (pinned); voice
  privacy section added to 07.
- **Tests (+35 → 381)**: FSM table (happy path, toggle-send, barge-in,
  cancel-at-every-stage, pause wins, error recovery, invalid moves,
  generation-bump rules); sentence chunking (boundaries, decimals, short
  fragments, newlines, stall guard, citation-only); WAV header; and a
  voiceService acceptance suite on the REAL service + sql.js + real bridge
  (quick ask generic-only + spoken, persistence opt-in with session reuse,
  Space grounding, muted/failed-synth text fallbacks, unsummoned
  contributions never spoken, barge-in stale-audio drop, cancel from every
  stage, no-speech/empty-STT soft failures, pause drops audio, in-session
  summon through the real engine, no audio in logs).

**346 tests green** (42 files), typecheck + build clean. No version bump.
