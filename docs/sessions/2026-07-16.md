Two things today: (1) an exhaustive hard test that proves v1.5.2's privacy fix
holds under every edge action, and (2) v1.5.3 — closing the last screen-capture

## Hard test — `scripts/privacy-affinity/hardtest.js`

A stricter superset of `drive.js`. Launches the built app (isolated userData,
CDP on) with a separate-process affinity probe reading each window's real
`GetWindowDisplayAffinity` at ~15ms, and drives **every** edge action that could
wipe capture-exclusion:

- nav / dropdown / hover storms, dashboard minimize/maximize/restore, overlay
  hide/show, privacy re-assert;
- a real live interview (loopback capture — the known wipe trigger) + the full
  Cue Card control storm (expand/compact, opacity, click-through, type/format
  sweeps, pronunciation, Ask box);
- an **OS-level move storm** (koffi `SetWindowPos`, ~130–160 moves) — the
  "hold-and-drag the Cue Card" test, at the OS level so it doesn't depend on
  synthetic-input hit-testing;
- region selector (hides+reshows both windows), pause/resume/stop.

**Result (3 runs): PASS.** Dashboard + Cue Card **100% protected of visible time,
0 sustained dips**; one run caught a single 31ms self-healed transient on the
briefly-maximized dashboard (still ≤ one share frame). The app logged 4–22 real
OS wipes/run (loopback start, selector, window churn), each healed off-thread
faster than one 15ms probe sample.

Harness gotchas baked in: app windows carry **empty OS titles** → label by SIZE
(Cue Card 440×460 / 520×680; anchor 320×240 is titled); the profile `Select` is
an in-window `Dropdown` (open via `[aria-haspopup="listbox"]`, click the option,
verify the "Interviews" `h3` appears — exact-match on the U+2026 placeholder is
fragile); env `OPENAI_API_KEY` makes `apiKeyPresent` true so Start enables; gate
the final-state check on visibility (a destroyed pre-reload window lingers
invisible at 0x0).

## v1.5.3 — in-window confirmations (last leak)

The one residual: the app's **native confirm dialogs are separate OS windows at
affinity 0x0**, so they show in a screen share even while the app is hidden —
worst of all "Turn off Privacy Mode?", which fires while privacy is still on.
Three convertible gates (all `dialog.showMessageBox`): `requestPrivacy`
(privacy.ts), `confirmQuit` (quit.ts), `confirmDestructive` (data.ipc.ts, reused
by settings reset).

**Fix — a main-driven in-window confirm** (confirmation stays in main, per the
existing "destructive actions re-confirm even if the IPC is called directly"
guarantee; also covers tray / global-shortcut callers with no renderer focus):

- shared: `IPC.ui.confirmResponse`, `EVENTS.confirmRequest`, `ConfirmRequest`.
- main `services/ui/confirm.ts`: `confirmInWindow(opts)` — pick a protected host
  window (focused app window if visible → visible main → visible overlay → show
  main; native `dialog` fallback only if no renderer window exists), send
  `confirmRequest`, await the reply; resolve `false` on timeout / host close.
  `registerConfirmIpc()` wires the reply handler.
- preload: `api.ui.confirmResponse` + `events.onConfirmRequest`.
- renderer `components/ConfirmHost.tsx`: subscribes and renders the shared
  `Modal` (Cancel + tone-colored Confirm), mounted once in `Root` (main.tsx)
  beside `TooltipShield`; wrapped in `data-ct-interactive`. Main targets one
  window, so only that window's host shows a modal.
- rewired the 3 gates to `confirmInWindow`, dropping `dialog.showMessageBox`.

**Left native (OS-owned, documented):** file pickers
(`showOpenDialog`/`showSaveDialog`) and the tray context menu.

**Tests:** `confirm.test.ts` (12 — host selection, reply→resolve, timeout,
host-close, native fallback); `privacy.test.ts` mocks `../ui/confirm` to stay
isolated. No RTL/jsdom infra in the repo, so `ConfirmHost` is covered
end-to-end by the hard test's new **confirm-leak check** instead: it triggers
each confirm, asserts an in-window modal appears in the DOM (not a native aff-0
window), then cancels. 204 unit tests green; typecheck + build clean; hard test
PASS with all three confirms `in-window ✓`.

`feat/v1.5.3-inwindow-confirms`, commit `89c0d55`.
