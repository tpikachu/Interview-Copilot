  default Electron icon. Tray refactored to reuse it. Note: in *dev* the Windows

- **Region selector** (`selectionWindow.ts`): create with `show:false` and reveal
  on `ready-to-show`/`did-finish-load`/1500ms fallback (no blank black flash);

  trigger isn't foreground, so plain show() could leave it behind). Added a

  window) instead of the stale "transparent" assumption.
- **Capture diagnostics** ([screenshot.ts](src/main/services/capture/screenshot.ts)):

  an empty capture so the selector never opens onto nothing.

  (non-black) 1920×1080 image on this machine (avgBrightness 31) — so the "blank
  selector" is NOT a GPU black-frame issue; it's the window/paint/foreground path,
  which the above fixes target. Restored `privacy_mode` to ON (a temp probe had
  flipped it; the force-kill discarded the write, so it was already ON).

### Region selector — root cause narrowed (reproducible report)

User repro: capture works first time(s); after the AI finishes an answer (overlay
shown), the next capture is "blank". Diagnosed with probes:

- **Capture is NOT the problem.** `desktopCapturer` returns a real (non-black)
  1920×1080 image every time — baseline, after a faked completed solve, and across
  5 open/close cycles (brightness ~31). So it's not a GPU black-frame.
- **The selector window isn't being raised to the foreground.** After the solve,
  the overlay (Cue Card) was active; opening the selector from a non-foreground
  state leaves it BEHIND the user's window → looks blank, can't drag. A headless
  screenshot confirmed the selector stays behind the dashboard (this is also just
  the headless foreground-lock, which can't be beaten without real input — so it
  can't fully verify the fix, only the diagnosis).
- **Fix:** `app.focus({ steal: true })` + re-assert always-on-top + `moveTop()` +
  `focus()` on reveal (selectionWindow.ts) to defeat the Windows foreground lock.
- Couldn't 100% verify headlessly (no real keypress to grant foreground); needs a
  real Ctrl+Shift+S test. If it persists, next options: minimize/restore foreground
  hack, or make the selector a child of the active window.

### RESOLVED — region selector blank (live debug with user)

Live session nailed it. Logs on the user's real capture showed:
`capture … empty=false avgBrightness=31` (capture fine) + `selector shown
(timeout): visible=true focused=true onTop=true` (foreground fix worked, window IS
on top) + `renderer dev server not ready at …?view=selection`. A user screenshot
showed a full-screen **black** window (taskbar only) — i.e. the selector was on top
but its React UI never loaded, so the timeout revealed an empty black window.

Root cause: creating the selector window **on demand, right after the GPU-heavy
screen capture**, made its renderer fail to load from the dev server (the dashboard
+ overlay, created at startup, load fine).

Fix: **pre-create the selection window at startup (hidden) and reuse it**, like the
overlay. `openSelector` now captures, pushes the frame via a new `selection:reset`
event + resets the renderer's drag state, repositions to the cursor's display, then
shows. `closeSelector` hides (doesn't destroy). Kept the foreground-steal + Esc
safety + capture diagnostics. **User confirmed it works.**

## v0.4.1 — Cue Card placement & persistence

- Cue Card defaults to the **bottom-right** of the primary display's work area
  (`EDGE_MARGIN` 24px) and is taller: compact `440×460`, expanded `520×680`.
- **Persist geometry**: new `SETTINGS_KEYS.overlayBounds`; `overlayWindow.ts`
  saves `getBounds()` (debounced 400ms) on `move`/`resize`, and restores it on
  creation via `initialBounds()` with an `isOnScreen()` guard (falls back to the
  bottom-right default if the saved monitor is gone). Added to `APP_SETTING_KEYS`
  so factory reset restores the default.
- `setOverlayMode` now anchors the bottom-right corner and clamps to the work
  area, so expanding doesn't push the card off-screen.
- Smoke-tested: clean boot, 0 shortcut-registration failures (earlier failures
  were a leftover dev instance holding the global shortcuts, not a regression).
