## Shipped (6 commits)

1. **Dev-infra pair** (`93206f8`) — e2e/fixtures.ts spawned `out/main/index.js`
   directly, so `app.getAppPath()` missed the repo-root drizzle migrations and the
   suite died on fresh userData ("no such table") — now spawns the app DIRECTORY.
   The electron-vite wrapper normalizes a lowercase Windows drive letter in cwd
   (the real cause of the "transient rollup build error" gotcha) and `npm run
   build` routes through it. CLAUDE.md updated.
2. **Realtime auto-reconnect** (`4da4580`) — the audit's #1 owed item. Bounded
   exponential backoff (500ms→8s ×5), a 30s-stable connection refills the budget,
   stale-socket guards, one specific error per cycle + one final actionable message
   on exhaustion. Cue Card shows a "reconnecting audio…" pill
   (`session:transcriber-status`). +6 tests (fake ws + fake timers).
3. **Unified schema wave** (`f5dc1ec`, migration 0007) — `sessions.kind`
   (live|mock|sparring), new `answer_feedback` table, ai_answers drops the dead
   expanded-meta columns (talking_points/resume_match/star/clarifying_question;
   meta plumbing slimmed to riskWarning), and **the FK-action rebuild**
   (`db/fkRebuild.ts`): migration 0001 created `chunks.job_id`/`sessions.job_id`
   with bare REFERENCES, so ON DELETE was "no action" vs the schema's promised
   CASCADE/SET NULL. SQLite can't ALTER an FK and DROP TABLE with FKs on would
   cascade-wipe embeddings → documented rebuild dance (FK off →
   create/copy/drop/rename → foreign_key_check), idempotent, run post-migrate.
   **Verified against a copy of the real dev DB** (actions fixed, row counts
   preserved, job-delete actually cascades, second run no-op).
4. **Practice Loop flagship** (`02629c3`) — Sparring already rated every spoken
   answer 1–5 with strengths/improvements/tip, then threw it away. Now each drill
   persists as a session (kind `sparring`): questions → detected_questions, spoken
   answers → transcript + `answer_feedback` rows written AS THEY HAPPEN;
   `evaluateAnswer` also classifies the probed **competency** (closed
   StoryCompetency set). Ending a drill assembles its report **locally** (no LLM):
   average + strongest/focus-next summary, deduped strengths/improvements,
   per-question `[n/5]` verdicts. `sessionsRepo.practiceStats()` → Reports
   **Practice card** (average stat, per-drill sparkline, per-competency bars —
   single-hue indigo, dataviz-skill checked) + "Practice" badges. Stray live
   drills self-heal; failed first question leaves no row; Interview page ignores
   non-live kinds for Start/Resume.
5. **Follow-up prediction** (`a3ab1b4`) — post-`answerDone`, a fire-and-forget
   classify-tier call predicts the interviewer's likely follow-up; persists to
   the kept `followup_question` column and renders as a "Likely follow-up:" line
   under the finished answer. Never touches first-token latency; skipped in mocks.
6. **Privacy Mode honesty on Linux** (`2cf3b9b`) — setContentProtection is a
   silent no-op there; Settings now warns explicitly and the Cue Card privacy
   button goes amber with an honest tooltip.

Verified along the way: `typecheck` · **173 unit tests** (+13 this release so far)
· `build` green at every commit.

## Adversarial review

3 dimensions → refute-verify over the full branch diff: **7 confirmed, 0 refuted**
(two reviewers independently found the same #1). All fixed in the same commit:

1. *(high)* **`sparringManager.answer()` deref'd the global `spar` after two
   multi-second awaits** — "End sparring" mid-evaluation (the button stays enabled
   during 'Evaluating…') threw a raw TypeError, and end+restart wrote the OLD
   answer's transcript/feedback into the NEW drill's session. Fixed with the
   entry-capture pattern: `const s = spar` + revalidation after every await before
   any DB write (applied to `ask()` too).
2. *(med)* **Reports fell back to the LLM generator for sparring rows without a
   report** (reachable via app-quit strays), fabricating "(no answer)" coaching
   from ai_answers rows sparring never writes. `generateReport` now branches on
   kind: sparring → `buildSparringReport()` (the extracted local assembler);
   zero-answer drills get a friendly error instead of a junk LLM report.
3. *(med)* **"reconnecting audio…" pill stuck forever after budget exhaustion** —
   the status union gains a terminal `'disconnected'`, emitted at exhaustion and
   on the mid-cycle no-key bail; the overlay already clears on any
   non-reconnecting status.
4. *(med)* **Stale follow-up prediction could annotate a regenerated answer** —
   a per-question generation counter (bumped at stream START, so even an aborted
   regenerate supersedes) gates the fire-and-forget's DB write + broadcast;
   cleared per session.
5. *(low)* **In-band Realtime server errors bypassed the once-per-cycle latch**,
   letting the generic exhaustion message clobber a specific quota error — the
   latch now covers both paths (+1 test asserting the last surfaced error stays
   specific).
6. *(low)* **Quitting from the Sparring page left the drill's row `status='live'`
   forever** (React cleanup doesn't run on window destroy), haunting the
   sidebar's "Interview live" counter across launches — `sparringManager
   .healStrays()` now also runs at app boot, and never touches the actively
   running drill.

Verified after fixes: `typecheck` · **174 unit tests** · `build` green.
