# 1.5.1 — 2026-07-15

A critical fix, shipped alone and fast.

## Fixed

- **Privacy Mode was silently failing on some Windows 11 systems.** On certain
  Windows 11 builds and graphics drivers, the graphics pipeline rendered app
  windows through a path that bypasses Windows’ screen-capture exclusion — so
  BrainCue could appear **fully visible in screen shares and recordings** even
  though Privacy Mode said it was hidden. The app now routes rendering through a
  path the exclusion filter reliably handles.

  Verified end-to-end against the same capture technology Zoom, Meet, and Teams
  use: with Privacy Mode on, BrainCue is now absent from the capture; with it
  off, it appears — exactly as promised.

  If you ever see rendering glitches after this update (rare, driver-specific),
  launch with `BRAINCUE_ALLOW_DIRECT_COMPOSITION=1` to revert — knowing Privacy
  Mode may be capturable on affected machines in that mode.

**Please update — and if you rely on Privacy Mode, do a 10-second check after
updating:** start a screen share in your meeting app and confirm BrainCue's
windows don't appear.
