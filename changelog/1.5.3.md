# 1.5.3 — 2026-07-16

**Privacy Mode: the app's confirmation dialogs no longer appear in a screen
share either.**

## Fixed

- **Confirmation prompts are now drawn inside the app, so they're hidden from
  screen shares like everything else.** A few actions ask you to confirm first —
  turning **off** Privacy Mode, **Exit**, and Settings → **Reset settings** /
  **Delete all data**. Those prompts used to be native operating-system dialogs,
  which are *separate windows* that Privacy Mode can't cover — so they flashed
  into a screen share for a moment even while the app itself was hidden. The
  worst case was the "Turn off Privacy Mode?" prompt, which by definition appears
  while Privacy Mode is still on. Every one of these confirmations is now a panel
  drawn inside the app's own window, so it's excluded from capture together with
  the rest of the app. Behavior is otherwise unchanged — the same wording, the
  same safe default (a destructive action is never the pre-selected button), and
  it still works when triggered from the tray or a global shortcut with the
  dashboard hidden (the window is surfaced to show the prompt).

  With this, no part of BrainCue — windows, dropdowns, tooltips, or
  confirmations — appears in a screen share while Privacy Mode is on. (Native
  file pickers for choosing/saving a file remain standard OS windows, as in every
  app; they show your file system, not app content.)

- New exhaustive **verification harness** (`scripts/privacy-affinity/hardtest.js`)
  that drives the built app through every edge action — navigation, dropdowns,
  a live interview, dragging the Cue Card around, opening the region selector,
  and each confirmation prompt — while a separate process reads every window's
  true Windows capture-exclusion state. It confirms the dashboard and Cue Card
  stay excluded from capture the entire time, and that each confirmation renders
  in-window rather than as a native dialog.
