# 1.5.2 — 2026-07-16

**Privacy Mode: the app no longer appears in a screen share once an interview
starts.**

## Fixed

- **The Cue Card and dashboard no longer become visible in your screen share
  when you start a live interview.** Root cause, found with ground-truth testing
  (reading each window's real Windows capture-exclusion flag,
  `GetWindowDisplayAffinity`, before/after every step): to transcribe the
  interviewer we capture **system (loopback) audio**, and on Windows the only way
  to get it requires a *video* capture alongside it. That video capture was of
  the **whole screen** — and Windows/Chromium responds by switching **off** the
  capture-exclusion on *all of this app's own windows* for as long as the screen
  capture is live (so an app sharing its screen can see its own windows). So the
  moment an interview started, both windows dropped out of stealth and **stayed
  visible** for the rest of the session. Everything before this — the flicker,
  the "still visible on interaction" — was the same cause seen through different
  workarounds (a periodic re-hide fought the capture and flickered; no re-hide
  left it steadily visible).

  The fix removes the cause: the loopback audio now captures a **tiny, parked
  off-screen window** instead of the screen. A window capture only clears the
  exclusion once at start (not continuously), so restoring it makes it stick.

- **Windows are hidden by watching the real protection state — not by blindly
  re-hiding on a timer.** Some screen-sharing and remote-desktop tools silently
  strip the app's capture-exclusion every few seconds while they're capturing
  (so the person on the other end isn't looking at a black rectangle) — exactly
  when it matters. Earlier builds re-applied the exclusion blindly (on clicks,
  moves, and short intervals), and every one of those re-applies could flash the
  window into the share for a frame — the "flashing" some users saw. The app now
  **watches the actual Windows exclusion flag on a background thread and
  re-applies it only when something has really removed it** — reading the flag is
  invisible, so when nothing is wrong, nothing flashes. When it *is* stripped,
  it's restored in about **1–2 milliseconds** (measured), far below a single
  frame of any screen share, using the lightest possible OS call. Verified
  end-to-end with a separate-process probe reading each window's true state
  through a full driven session (navigation, dropdowns, a real live interview,
  Cue Card clicks): the dashboard and Cue Card were never capturable for a single
  sample.

- **Typing in the Cue Card's "Ask a question" box works again.** The previous
  release made the Cue Card a non-activating window to chase stealth; that also
  stopped it from taking keyboard focus, so the Ask box couldn't be typed into
  (especially with a meeting app in the foreground). Now that stealth no longer
  depends on it, the Cue Card is a normal window again — clicking, dragging,
  every control, and typing all behave normally.

- **Dropdown menus no longer leak into screen shares — including the dashboard.**
  A native dropdown's option list opens as a *separate* OS popup window that
  doesn't inherit the app's capture exclusion, so it showed in your share even
  while the app was hidden (e.g. picking a profile on the Interview page). Every
  dropdown in the app — the Cue Card's controls **and** all the dashboard
  selectors (profile, interview, type, voice, …) — is now an in-window popover
  that's hidden together with its window.

- **Hover tooltips no longer leak into screen shares.** A native tooltip (from an
  element's hover text) also renders as a *separate* OS window with no capture
  exclusion — so hovering the sidebar logo, a nav item, or any button flashed the
  tooltip into your share even while the app itself was hidden. All tooltips
  across the app and Cue Card are now drawn inside the window's own surface, so
  they're hidden together with it. (This was the leak you'd see just from moving
  the mouse around the dashboard.)

- New ground-truth verification harness (`scripts/privacy-affinity/`): drives
  the real app through a full session while a separate process reads every
  window's true Windows capture-exclusion state at ~15ms resolution.

> **Note on the dashboard window:** it stays hidden by the same watcher —
> all app windows are covered, all the time, whether or not an interview is
> running.

**After updating, do a quick check:** share your screen in a meeting app, start
an interview, then click and drag the Cue Card and open the Type dropdown —
confirm none of it appears in the shared view.
