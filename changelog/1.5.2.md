# 1.5.2 — 2026-07-15

**Privacy Mode: the Cue Card is now truly invisible in screen shares — no
flashing, ever.**

## Fixed

- **The Cue Card no longer appears in a screen share when you click, drag, or
  interact with it — and there is no more flickering.** The root cause, found
  with ground-truth testing (a separate capture process using the same Windows
  capture technology as Zoom/Meet/Teams, driven by real mouse input): on this
  class of Windows 11 build, **activating a window** — bringing it to the
  foreground by clicking it — is what makes Windows silently stop hiding it from
  screen capture. Worse, the only way to re-hide it is to re-apply the
  exclusion, and **that re-apply itself causes a one-frame flicker in a live
  screen share** — so any approach that keeps re-hiding the window produces the
  flashing you saw (the more often it re-hid, the worse the shimmer).

  The fix removes the problem instead of papering over it: **the Cue Card is now
  a non-activating window.** It still receives every click — buttons, the Type
  and Format controls, dragging, and the settings sliders all work exactly as
  before — but clicking it never brings it to the foreground, so Windows never
  drops the capture exclusion, so nothing ever needs to be re-hidden, so there
  is **nothing to flicker.** Verified against ground-truth capture through
  thousands of rapid clicks, format-button spam, dropdown toggling, window
  drags, and z-order changes: **zero frames leaked, zero re-applies, zero
  flicker.**

  One consequence of a non-activating window: it doesn't take keyboard focus on
  its own. The Cue Card's **"Ask a question" box** now grants itself the
  keyboard the moment you click into it and hands it back when you click away,
  so typing still works. (During those few seconds of deliberate typing the card
  behaves like a normal window; every other interaction stays perfectly hidden.)

- **Dropdown menus no longer leak into screen shares.** Native dropdown lists
  (the Cue Card's Type selector and the settings selects) open as *separate OS
  popup windows* that never inherit the app's capture exclusion — so the open
  list was visible in your share even while the app itself was hidden. All Cue
  Card dropdowns are now in-window popovers, hidden together with the window
  (verified: opening them creates no OS popup window at all).

- `scripts/verify-privacy-capture.mjs` now really toggles Privacy Mode for its
  before/after captures (it was silently calling a settings key that doesn't
  exist).

> **Note on the dashboard window:** the main dashboard still needs the keyboard
> (it's full of forms), so it can't be non-activating. It keeps a lightweight,
> event-driven re-hide. During a live interview you work from the Cue Card and
> the dashboard is normally minimized to the tray, so this doesn't affect
> stealth in practice.

**After updating, do a 10-second check:** share your screen in a meeting app,
then click the Cue Card, drag it, spam the Format buttons, and open the Type
dropdown — confirm none of it appears in the shared view, with no flicker.
