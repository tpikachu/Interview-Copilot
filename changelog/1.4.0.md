# 1.4.0 — 2026-07-13

This release refines the **live interview experience** — the Cue Card keeps the whole
conversation, pronunciation help moves into the answer itself, and coding answers follow
a real interview narrative — plus a full-project audit hardening pass under the hood.

## Added

- **Question history by default.** The Cue Card now keeps every question of the session:
  when a new question arrives, older cards collapse to a single line — click any of them
  to expand and re-read. No more losing the previous answer mid-conversation.

- **Inline pronunciation.** Hard-to-say words are annotated **right where they appear** in
  the answer (e.g. “Kubernetes *(koo-ber-NET-eez)*”) instead of a separate guide below —
  read straight through, no eye-jumping. Copied and saved answers stay clean.

- **Coding answers that talk like an interview.** Coding solutions now follow the
  four-beat story interviewers expect: **understand the problem → plan the approach →
  the code → efficiency evaluation** — and the delivery adapts to your selected answer
  format (terse beats for Key Points, a natural spoken walkthrough for Explanation and
  Story Teller, full depth for Detailed).

- **Answer application questions later.** Tailoring no longer forces you to provide the
  application questions up front — open any saved application and add questions (one per
  line) whenever they show up in the form; answers stay grounded in your tailored resume.

## Changed

- **Interview types are SWE-focused.** Product and Sales session types are gone —
  BrainCue is built for software engineers: Behavioral, Technical, Coding, System Design,
  and General.

## Under the hood

A whole-project audit (plus adversarial review of every fix) hardened the live pipeline:

- Concurrent answer streams (a live answer + a coding solve) can no longer interleave
  text into the same card; new coding solves cleanly cancel the previous one.
- A stale question-classification can no longer cancel an answer you asked for manually,
  and mock interviews recover cleanly when the first question fails or you stop early.
- Overriding the answer task to a reasoning model (GPT-5 / o-series) now works — with
  token headroom and a clear error instead of a silently blank card. Chat-tuned GPT-5
  variants are routed correctly as non-reasoning.
- Model settings saved from the dashboard and the Cue Card no longer overwrite each
  other; resume-tailoring calls get a hard timeout instead of silent retries.
- Fetching a job posting or company site by URL is now bounded in size and time (a huge
  or hostile page can’t exhaust memory), and redirects can’t steer the fetch into
  local/private addresses.
- The unit-test suite grows to 161.
