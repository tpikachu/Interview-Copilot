# 2.0.0 — 2026-07-22

**BrainCue 2.0 — from interview copilot to the AI that's in the room with you.**
One conversation engine now powers every mode; the app gained a voice, a memory
you approve, and an ambient companion — and the project opened up, with a
public site and an automated evaluation pipeline for contributors.

## Added

- **Modes over one engine.** Interview Copilot is now one preset of a shared
  conversation engine — *listen → decide when to contribute → ground in your
  documents → respond* — rather than the whole product. Modes that aren't ready
  yet appear in the start flow as **Coming soon**, so you can see where the app
  is headed.
- **Meeting Copilot (Labs).** Sits in a meeting quietly and surfaces context,
  unanswered questions, action items, and decisions. You choose its presence —
  Summoned only, Quiet, Balanced, or Active — and get a meeting report at the
  end.
- **Companion (Labs).** An ambient presence with memory while you work. Its
  posture is yours to set (Off / On demand / Assistive / Proactive), and it
  lives behind deterministic gates: quiet hours, per-session budgets, and a
  live cost meter that shows what the session is spending.
- **A voice of its own.** Answers can be spoken aloud with streamed,
  sentence-level speech — start talking over it and it stops to listen.
  **Push-to-talk from anywhere** (Ctrl+Shift+T) summons it even without a
  session, for a quick spoken answer.
- **Memory you approve.** BrainCue can remember things across sessions, but
  nothing is kept silently: it proposes memories, and only ones you approve are
  ever recalled. Review, correct, or forget any of them at any time.
- **One start flow for everything.** Every mode starts from the same launcher,
  with a transparency panel stating exactly what will be captured and what
  leaves your machine — before anything starts.

## Changed

- **The app is now BrainCue** (formerly AI Interview Assistant). Existing
  installs keep every profile, session, and document — the app pins its
  original data directory, so the rename never touches your data.
- **Mode-first dashboard.** Home is a launcher of mode cards with a live
  capture-status row; Library, Sessions, Insights, and Settings organize
  everything else.
- **Settings grew up.** Cost/quality model presets per task (Balanced,
  Low cost, Best) with per-task overrides, and a Providers card that shows
  which capability runs on which provider — with more providers on the way.
- **The onboarding tour** was rewritten for the mode-first app.

## Fixed

- **Fresh installs no longer lose the sessions "mode" column.** A
  post-migration table rebuild carried an outdated schema and silently dropped
  the column on brand-new installs, breaking session history until the next
  release. A regression test now guards every column.
- **Dropdowns behave.** Clicking outside (or switching windows) closes them,
  and choosing an option no longer reopens the list.
- **The Cue Card's last native dropdown is gone.** Native option popups are
  separate OS windows that Privacy Mode can't hide from a screen share; every
  picker now renders in-window, covered by capture exclusion like everything
  else.

## For contributors

- **BrainCue is now built in the open**: a public site and docs at
  <https://tpikachu.github.io/BrainCue/>, full governance (CONTRIBUTING,
  security policy, issue and PR templates), and a difficulty-tiered bounty
  backlog.
- **Every PR gets an automated scorecard**: instant intake gates, a secret
  scan, unit coverage of the changed lines, and a rubric-pinned AI review —
  with the rubric's hash embedded in every scorecard so the standard can't
  move silently. Merges are issue-first and armed to complete automatically on
  maintainer approval. The pipeline is designed for listing on Bittensor's
  GitTensor subnet, where merged PRs earn contributors TAO.
