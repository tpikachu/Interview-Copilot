# 0.9.0 — 2026-06-25

Smarter coding help and finer control over models: a dedicated reasoning model for
coding problems, multi-screenshot capture for long ones, cost/quality presets with a
live picker, answer history, a listen-only coding mode, and a round of reliability fixes.

## Added

- **A dedicated coding solver.** Coding problems now route to their own model — a
  reasoning model by default — used by **both** "Solve from clipboard" and "Solve a
  region." It's the one place a slower, smarter model earns its cost, while your live
  answer cues stay fast.
- **Pick the coding model live, per problem.** A new **Coding solver** section in the
  Cue Card settings lets you switch the model and **reasoning effort** on the spot —
  bump a hard problem up to a stronger model, keep an easy one cheap.
- **Capture long, scrolling problems.** A LeetCode-style problem rarely fits one
  screen, so the **Region** button now **accumulates screenshots**: scroll, capture,
  repeat — a thumbnail strip shows what you've grabbed — then **Solve** sends them all
  at once and the model reconstructs the full problem. Captured at higher fidelity so
  code stays legible.
- **Model presets.** **Settings → OpenAI Models** now offers **Balanced** (default),
  **Low cost**, and **Best**, plus a **Custom** indicator that lights up automatically
  when you override any individual task. Per-task overrides always win on top.
- **Answer history (📚).** Toggle it on and new answers no longer replace the old —
  past answers **collapse** above the current one, expand on click, and each is
  **removable** individually.
- **Listen-only coding mode.** In a **Coding** interview the interviewer is still
  transcribed, but BrainCue **won't auto-answer** by default — so a coding answer you're
  reading isn't wiped when they speak. Flip **🎧 Answering** on to answer what they just
  asked (and resume auto-answers).
- **Errors are now visible.** A failed microphone/screen-audio grant or a dropped
  transcription connection surfaces a clear message in the dashboard and the Cue Card,
  instead of silently showing a "listening" state forever.

## Changed

- Coding screenshots are sent at full detail and captured at higher resolution, so
  small editor text reads reliably.
- Choosing a preset is a clean switch — it clears per-task overrides so the
  configuration matches the preset (no lingering "Custom").

## Fixed

- Cancelling the screen-audio picker or denying the microphone no longer leaves a
  phantom "live" session with nothing flowing — audio is acquired before the session
  starts.
- Two back-to-back transcript turns can no longer double-answer the same question.
- A malformed audio frame can no longer crash the app while driving the Cue Card meter.
- Mock rehearsals no longer leak audio or leave an orphaned session when you navigate
  away mid-run.
